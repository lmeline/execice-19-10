Tutoriel Git pour les débutants

Le développement collaboratif de projets de logiciels ne se produit pas uniquement au sein des entreprises. Dans le secteur open source, plusieurs centaines, voire milliers de bénévoles (selon la taille du projet) sont impliqués dans la maintenance, l'optimisation, le développement ou la modification d'un programme. Ce type de projets serait pratiquement impossible à réaliser sans un système adapté pour enregistrer et gérer les nombreuses modifications soumises par les différents développeurs.

L'une des solutions les plus populaires en matière de gestion des versions est l'outil sans licence Git. C'est facile à apprendre et gratuit. Dans ce didacticiel, nous expliquons les bases de Git les plus importantes dont vous avez besoin pour avec le programme de gestion de versions.

Qu'est-ce que Git ?

Git est un système de contrôle de version distribué développé par le créateur de Linux Linus Thorvalds en 2005 et publié sous la licence gratuite GNU-GPLv2. La particularité de cet outil est que, même s'il existe un référentiel central pour chaque projet, tous les utilisateurs impliqués téléchargent unecopie de travail locale de ce répertoire sur leurs propres appareils de travail. Chacune de ces copies représente une sauvegarde complète du référentiel, rendant inutile une connexion réseau continue et permettant un travail hors ligne. De plus, les copies peuvent être utilisées pour restaurer chaque projet en cas de panne ou de dommage du référentiel principal. modifications mises en œuvre peuvent être échangées à tout moment tous les autres participants au projet et – le cas échéant – incluses dans le référentiel.
